CREATE

新しい一冊を作る

Claudeに次の合言葉を送ると、いつもの体裁で記事を起こし、この書架にも自動で並べます。

/create [書籍名 または 学習内容] [--push]
  1. 必要に応じて「今のレベル / 目標レベル」を選択式で確認
  2. pages/<棚>/ に新しい記事HTMLを生成
  3. この書架の ARTICLES と catalog.json に1件追記
  4. --push を付ければそのまま公開まで

※ 実行時は最新の index.html を一緒に渡してください。

UPDATE

既存の一冊を直す

記事を手直ししたいときはこちら。変える所・変えない所を先に確認してから編集します。

/update [識別ワード or file名] [変更内容] [--push]
  1. catalog.json で対象記事を一意に特定（例: 「sql指南書」）
  2. 変更する所・しない所を先に明文化、曖昧ならヒアリング
  3. 意図しない箇所は触らず外科的に編集

※ ほかに /org（整頓）・/publish（公開）も。詳しくは README。

この書架は単一の index.html。蔵書データは末尾の ARTICLES 配列に格納し、1件追記するだけで棚に並びます。検索・分野フィルタは配列が増えても自動対応。記事本体は pages/ 配下、相対リンクなので GitHub Pages にそのまま置けます。

分野：データの土台 / バックエンド&API / ソフトウェア設計 / クラウド&インフラ / 分散データシステム / 開発ツール
